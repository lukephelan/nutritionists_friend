Technical Requirements

Create documented wireframes before creating the app
Interact with an API with HTTParty
CRUD with an SQL database
Create a 10 minute presentation
Create at least 5 RESTful routes with POST, PUT, GET and DELETE verbs using a Ruby routing library (Rails)
Use an ORM to create a database table structure and interact with data using Ruby
Have some form of login for the user
Properly manage packages using Gems
Have semantically clean HTML and CSS
Deploy to the public with something like Heroku

App-Specific Requirements

A website for a nutritionist to interact with their clients.

Clients can sign-up and log in and out
Users have their own interactive dashboard
Users can log their meals with a time and date stamp
Connect to a nutriton API to retrieve nutritional information about their meals
Dashboard displays their target macro-nutrient intakes (a max and min for each day)
Dashboard displays a few target micro-nutrient intakes (a min for each day)
They see a few graphs showing where they are for today in relation to their daily targets
Another few graphs shows their achievements over the past week (bonus points if I can get it to display a custom time range)
Nutritionist gets a different dashboard to the rest
The Nutritionist can select from a drop-down, each client to see their progress
They can also make comments on days or meals
The user can also make comments on days or meals
The Nutritionist is the one who sets the targets for their clients
Bonus points if I can make the targets set “from this point on” rather than be global across time (because goals change)
User and nutritionist can upload their profile photos
Users can input their body weight each day and show it against a target (this should be optional?)
Users can delete or edit log entries in case they make a mistake
Site use of the database on the website in the footer
